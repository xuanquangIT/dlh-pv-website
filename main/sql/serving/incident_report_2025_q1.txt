INCIDENT 001 — DATA PIPELINE OUTAGE
--------------------------------------
Date: 2025-01-14
Duration: 09:15 to 14:42 AEST (5h 27min)
Affected facilities: All 8 facilities
Severity: High
Status: Resolved

Description:
The hourly ingestion pipeline stalled during the Silver layer transformation stage.
Root cause was a PostgreSQL deadlock caused by concurrent writes from two pipeline
instances triggered by a failed retry mechanism.

Impact:
- 5 hours of missing energy readings across all facilities
- lh_silver_clean_hourly_energy had 320 null rows for the affected window
- Downstream forecast model received stale features, increasing 6h MAPE from 6.7% to 14.2%
- Alert was triggered at 09:47, 32 minutes after actual start

Resolution:
- Killed duplicate pipeline process
- Reprocessed missing window from Bronze layer
- Added distributed lock to prevent concurrent pipeline runs
- Backfilled 320 rows successfully by 16:00 AEST

Follow-up actions:
- Pipeline lock mechanism deployed in main branch (PR #47)
- Alert threshold tightened from 45-minute gap to 20-minute gap

INCIDENT 002 — YATPOOL SENSOR ANOMALY
----------------------------------------
Date: 2025-02-03 to 2025-02-07
Duration: 5 days
Affected facilities: Yatpool (YATSF1)
Severity: Medium
Status: Resolved

Description:
Yatpool reported shortwave_radiation values of 0 W/m2 during peak hours (09:00-15:00)
while energy_mwh readings were non-zero. This indicates a sensor failure on the
pyranometer rather than actual zero irradiance.

Impact:
- quality_flag set to SUSPECT for 120 hourly rows
- completeness_pct dropped to 71.4% for Yatpool across the 5-day window
- GBT model feature pipeline flagged is_valid = False for these rows
- Forecast accuracy at Yatpool declined: R2 dropped from 0.905 to 0.812

Data handling:
- Affected rows were retained with quality_flag = SUSPECT
- lh_silver_clean_hourly_weather rows marked is_valid = False
- Forecast model excluded these rows from feature calculation and used lag features

Resolution:
- Pyranometer replaced on 2025-02-07
- Sensor restored to normal operation on 2025-02-07 14:00 AEST
- Retrospective correction not applied: original SUSPECT-flagged data preserved
- Data quality score recovered to 100% by 2025-02-08

INCIDENT 003 — DUST STORM: EMERALD AND YATPOOL
-------------------------------------------------
Date: 2025-03-19
Duration: 12:00 to 23:00 AEST (11 hours)
Affected facilities: Emerald (EMERASF), Yatpool (YATSF1)
Severity: Low (expected weather event)
Status: Closed

Description:
Severe dust storm event in Queensland and western NSW regions.
AQI values reached 287 at Emerald and 241 at Yatpool between 14:00 and 19:00 AEST.
This is expected to reduce panel efficiency through soiling and reduced irradiance transmittance.

Impact:
- Emerald energy output: reduced by 38% compared to clear-sky model estimate
- Yatpool energy output: reduced by 29% compared to clear-sky model estimate
- aqi_value flagged as OUT_OF_BOUNDS (threshold: 200) in quality checks
- quality_issues column shows aqi_value_OUT_OF_BOUNDS for 22 rows per facility

Data handling:
- All rows retained with OUT_OF_BOUNDS flags
- Energy readings are valid (actual measured output, not sensor error)
- GBT v4.2 partially accounted for AQI impact: forecast was 18% below actual
  (model trained on AQI up to 200; extrapolation beyond that range less accurate)

No corrective action required. Weather event documented for model retraining consideration.

INCIDENT 004 — NETWORK REGION API TIMEOUT
-------------------------------------------
Date: 2025-03-28
Duration: 22:00 to 22:45 AEST (45 min)
Affected facilities: Bomen (BOMENSF), Finley (FINLEYSF)
Severity: Low
Status: Resolved

Description:
Network API endpoint for NSW1 region returned HTTP 504 timeouts for 45 minutes.
Bronze layer ingestion for Bomen and Finley had 3 missing 5-minute intervals.

Impact:
- 3 rows missing in Bronze layer for each facility
- Silver layer intervals_count = 9 instead of 12 for the 22:00 hour
- completeness_pct = 75.0% for that single hour
- quality_flag = INCOMPLETE for 2 rows

Resolution:
- Rows marked INCOMPLETE in Silver layer
- Bronze data not recoverable from API (historical window expired)
- Accepted as permanent data gap; included in quarterly quality report
